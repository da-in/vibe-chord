# Step 08. "어떻게 하면 ~" 질문 만들기

> 목적: 서비스 방향을 맞춘 뒤, **더 구체적인 생각**으로 수렴한다.

> **Guide:**  
> 1. Agent가 `docs/03`, `docs/05`, `docs/07`을 읽고 **"어떻게 하면 ~ 할 수 있을까?"** 형태의 질문을 만든다. 직접 질문을 추가해도 좋다.
> 2. 팀이 질문을 보며 답을 이야기한다.
> 3. 각 질문 아래 **답**을 적는다. (타이머 ON)  
> - 질문은 **많을수록** 좋고, 하고 싶은 말이 많은 질문이 좋다.  
> - **실제로 어떻게 할지** 떠올릴 수 있는 질문을 한다. (예: 사용자를 어떻게 모을까? → 너무 넓음)  
> - 답은 **질문과 같은 말 반복·뭉뚱그린 표현 금지**. 구체적인 문장으로 쓴다.  
>   - (X) 어떻게 하면 사용자가 이 기능을 더 잘 쓸까? → 사용자들이 더 쉽게 쓰게 만들면 될 것 같아요  
> - 참고한 서비스·자료가 있으면 답에 적는다.

**프롬프트 예시**

```
docs/03_service_goal_target_value.md, docs/05_pain_wow_points.md, docs/07_word_cloud.md를 읽고
docs/08_how_might_we_questions.md의 "Agent가 만든 질문" 섹션만 채워줘.

질문 규칙:
- "어떻게 하면 ~ 할 수 있을까?" 형태
- docs/03 목적·핵심 가치, docs/05 불편한 점·좋았던 점, docs/07 키워드를 골고루 반영
- 실제 방법·아이디어가 떠오를 수 있는 질문 (너무 넓은 "사람 모으기"류 X)
- 최소 10개 이상, 서로 겹치지 않게
- 각 질문 아래 답 칸은 비워둬 (팀이 채움)
- "팀이 추가한 질문" 섹션은 건드리지 마
- 구조와 상단 Guide는 유지
```

## Agent가 만든 질문

> `docs/03`(목적·가치), `docs/05`(불편한 점·좋았던 점), `docs/07`(키워드)을 바탕으로 Agent가 채운다.

### 질문 1: 어떻게 하면 코드·진행을 고르거나 이어 붙이는 **즉시** 소리가 날 수 있을까?

- 코드 블록·후보 카드를 **클릭·탭하는 순간** 브라우저 **Web Audio API**(또는 Tone.js)로 해당 코드를 **0.3초 이내** 재생한다. 별도 "재생" 확인 단계 없음.
- 타임라인에 코드를 **드래그해 추가**할 때도, 놓는 즉시 **그 코드 1마디**가 자동 재생되게 한다. (GarageBand 루프 즉시성 참고)

### 질문 2: 어떻게 하면 **Cmaj7·I-IV-V** 같은 기호를 몰라도 **느낌·분위기**만으로 코드 후보를 고를 수 있을까?

- 첫 화면에 **"밝은 / 슬픈 / 몽환적 / 긴장감"** 같은 **분위기 칩**을 보여 주고, 칩 선택 시 해당 분위기에 맞는 **시작 코드 3~4개**만 카드로 제시한다. 카드에는 **기호 대신** "잔잔한 시작", "희망적인 시작" 같은 **한국어 라벨**을 쓴다.
- 고급 모드(코드명 표시)는 **기본 숨김**하고, 원하면 토글로 Am·Cmaj7을 볼 수 있게 한다. (Hookpad는 기호 중심 → 우리는 **느낌 라벨 우선**)

### 질문 3: 어떻게 하면 코드 하나를 고른 뒤 **다음 후보 3~4개**를 **들으며** 비교·선택할 수 있을까?

- 마지막으로 고른 코드 오른쪽에 **"다음 코드 추천" 패널**을 띄우고, 후보 3~4개 각각 **▶ 미리듣기** 버튼을 둔다. 미리듣기는 **직전 코드 → 이 후보**로 **2코드만** 짧게 이어 재생한다.
- 후보 중 하나를 **클릭하면** 타임라인에 추가되고, 추가된 순간 **전체 진행 마지막 2코드**를 자동 재생해 "붙였을 때" 감각을 확인한다.

### 질문 4: 어떻게 하면 AI·추천 목록 **밖**에서 **전체 코드·진행 목록**을 열어 직접 실험할 수 있을까?

- 화면 하단 또는 사이드에 **"전체 코드 열기"** 버튼을 두고, **24개 diatonic 코드**를 **색·분위기 태그**(밝음/어두움)와 함께 그리드로 보여 준다. 그리드에서 탭하면 **즉시 소리 + 타임라인 추가** 가능.
- **"유명 진행 프리셋"**(카논 진행, 4코드 팝 등) 목록도 별도 탭으로 제공하고, 프리셋 선택 시 타임라인에 **통째로 불러온 뒤** 마디·코드를 **하나씩 바꿔** 실험할 수 있게 한다.

### 질문 5: 어떻게 하면 만든 코드 진행(4~8마디)을 **Play 한 번**으로 처음부터 끝까지 재생할 수 있을까?

- 상단 고정 **▶ Play / ■ Stop** 버튼 하나로, 타임라인의 코드를 **BPM 80~120**(기본 90) **4/4**로 **순차 재생**한다. 재생 중 **현재 코드 블록**을 하이라이트한다.
- 마디 수(4 / 8)와 **코드당 박 수**(1코드=1마디 vs 2박)를 **슬라이더 2개**로만 조절 가능하게 해, DAW 없이도 **길이감**을 맞출 수 있게 한다.

### 질문 6: 어떻게 하면 코드 진행이 **타임라인·블록**처럼 보여서 "이어 붙이기"가 직관적으로 느껴지게 할 수 있을까?

- 가로 **타임라인**에 코드를 **색 블록**(분위기별 색)으로 나열하고, 블록 **사이 "+"** 를 눌러 다음 코드를 끼워 넣는 구조로 한다. (DAW 트랙보다 단순 — **코드 줄 하나**만)
- 블록을 **드래그**해 순서를 바꾸면, 놓는 즉시 **바뀐 순서로 Play** 미리듣기가 돌아가게 한다. 블록 **길이**는 마디 단위로만 조절(복잡한 편집 없음).

### 질문 7: 어떻게 하면 **첫 방문 10분 안에** 유튜브·이론 없이 "내가 고른" 코드 진행 하나를 만들게 할 수 있을까?

- onboarding **3스텝만**: ① 분위기 고르기 → ② 시작 코드 1개 탭 → ③ 추천에서 다음 코드 **2~3번** 탭 → **Play**. 회원가입·DAW 용어 설명 **생략**.
- **"4마디 완성"** 배지를 4번째 코드 추가 시점에 띄우고, 그 순간 **자동 Play** + "첫 진행 완성!" 메시지로 **완성 경험**을 준다.

### 질문 8: 어떻게 하면 추천 후보마다 **"이럴 때 고르면 좋다"**(에너지 올리기, 잔잔히 마무리 등)를 **한 문장**으로 보여줄 수 있을까?

- 각 후보 카드 아래에 **20자 이내** 안내 문구를 고정 표시한다. 예: "에너지를 올리고 싶을 때", "브릿지처럼 잠깐 어둡게", "대표적인 마무리 느낌".
- 추천 로직은 **화성학 규칙(다이아토닉·cadence)** 기반이되, UI에는 **cadence·dominant** 같은 용어 대신 위 **상황 문구**만 노출한다. (필요 시 ℹ️ 누르면 "왜?" **접기**)

### 질문 9: 어떻게 하면 **보컬만** 하거나 **악기가 없는** 사용자도 작곡·탐색을 시작할 수 있을까?

- 모든 코드 재생을 **내장 피아노·기타 MIDI 톤**으로 통일하고, **악기 연주·입력 UI는 없음**. 클릭·탭·드래그만으로 작곡.
- **마이크·MIDI 키보드 연결 없이** 동작하며, 모바일·태블릿에서도 **한 손 조작**으로 후보 듣기·추가가 가능한 **큰 카드 UI**를 쓴다.

### 질문 10: 어떻게 하면 **친구·지인**에게 코드를 부탁하지 않고도 **"오, 이 느낌!"** 순간을 스스로 만들 수 있을까?

- **추천 + 즉시 재생**으로 "친구가 쳐 줬을 때"와 같은 **청각 Wow**를 **혼자** 반복 경험하게 한다. 후보를 바꿀 때마다 **2초 안에** 다시 들을 수 있어 **스스로 만족**할 때까지 실험.
- **되돌리기(Undo)** 한 번으로 직전 코드 선택을 취소해, "친구한테 다시 물어보기" 대신 **부담 없이** 여러 번 시도하게 한다.

### 질문 11: 어떻게 하면 ChatGPT처럼 **텍스트 코드표만** 주는 게 아니라, **만들면서 바로 듣게** 할 수 있을까?

- (ChatGPT Pain 대응) UI **기본 화면에 코드표 텍스트 목록을 두지 않고**, 모든 선택·추천 결과를 **재생 가능한 블록**으로만 보여 준다.
- 자연어 입력(선택)이 있더라도, 응답은 **"Am-F-C-G" 문자열**이 아니라 **타임라인에 4블록 추가 + 즉시 Play** 형태로 반환한다.

### 질문 12: 어떻게 하면 **코드 진행만** 있어도 밴드·동아리에 **"첫 자작곡 데모"**로 들려줄 **완성감**을 줄 수 있을까?

- Play 시 **기본 드럼·베이스 패턴**(아주 단순 4/4)을 코드와 **함께** 재생해 "반주가 있는 데모"처럼 들리게 한다. (루프만 있는 GarageBand와 달리 **내가 고른 코드**가 중심)
- **"데모 공유"** 버튼으로 **30초 mp3** 또는 **공유 링크**를 생성해, 밴드 단톡방에 **코드 진행 + 간단 반주**를 바로 보낼 수 있게 한다. 멜로디·가사 없어도 **"우리 곡 코드 이렇게 가자"** 회의에 쓸 수 있음.
