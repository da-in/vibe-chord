# Step 11. 고객 여정 맵

> 목적: 팀이 **우리 서비스**에서 사용자가 **어떤 순서로 무엇을 하는지** 같은 그림으로 이해한다. (Step 04는 기존·대체 방식, Step 11은 우리 서비스)

> **Guide:**  
> 1. Agent가 `docs/10_page_routing_split.md`만 읽고 **고객 여정 맵 초안**을 작성한다.  
> 2. 팀이 표를 보며 단계·행동·감정이 맞는지 이야기하고 수정한다.  
> - **데모에서 보여줄 핵심 여정 1개**를 중심으로 쓴다.  
> - Step 10의 **페이지·기능·이동 흐름**을 바탕으로 단계를 구성한다.  
> - 각 단계는 **구체적인 행동**으로 쓴다. (예: "앱을 연다" → "일정을 입력하고 추천 버튼을 누른다")  
> - Step 04처럼 **표 한 벌**로 정리한다. 팀원 모두가 흐름을 따라갈 수 있어야 한다.

**프롬프트 예시**

```
docs/10_page_routing_split.md만 읽고 docs/11_customer_journey_map.md를 채워줘.

규칙:
- Step 10의 페이지 구성·기능 목록·페이지 간 이동 흐름만 참고
- 우리 서비스 기준의 핵심 여정 1개 (데모 가능한 흐름)
- 단계 | 사용자 행동 | 생각/감정 | 해당 페이지 | 사용 기능(F-xx) 표 작성
- Step 10에 있는 페이지와 F-xx를 빠짐없이 연결
- 구조와 상단 Guide는 유지
```

## 여정 요약

> **누가** → **무엇을 하려고** → **어떤 순서로** → **무엇을 얻는다**

- **누가:** 화성학·DAW를 모르는 **완전 초보** (보컬, 초보 밴드, 학교·동아리)
- **무엇을 하려고:** 동아리 **첫 자작곡**을 위한 **코드 진행**을 스스로 만들고, 밴드에 **데모**로 들려주려 함
- **어떤 순서로:** 온보딩(분위기·시작) → 스튜디오(추천·듣기·Play) → (선택) 탐색 → 데모 공유
- **무엇을 얻는다:** **"내가 고른" 코드 진행 4~8마디** + **Play로 들은 완성감** + **밴드에 보낼 mp3·링크**

## 고객 여정

> MVP·데모에서 보여줄 주 사용자 흐름 — **첫 방문 → 첫 코드 진행 완성 → 밴드 공유**

| 단계 | 사용자 행동 | 생각/감정 | 해당 페이지 | 사용 기능 |
|---|---|---|---|---|
| 1. 진입 | 링크·QR로 서비스 접속. 회원가입 없이 **온보딩** 화면이 바로 열린다 | "가입도 없네, 일단 해보자" — 부담 ↓ | `/onboarding` | F-35 |
| 2. 분위기 선택 | **"밝은 / 슬픈 / 몽환적 …"** 분위기 칩 중 하나를 탭한다 | "우리 밴드 곡 분위기랑 비슷한 걸로" | `/onboarding` | F-03, F-34 |
| 3. 시작 코드 고르기 | 제시된 **시작 코드 카드 3~4개**를 탭하며 **즉시 소리**를 듣고, 마음에 드는 **"잔잔한 시작"** 카드를 고른다 | "글자(Am)는 몰라도 **소리로** 고를 수 있겠다!" (Wow) | `/onboarding` | F-04, F-05, F-01, F-34 |
| 4. 온보딩 마무리 | 온보딩 안내에 따라 **추천 후보 2~3개**를 더 탭해 코드를 이어 붙이고, **Play**로 짧게 확인한다 | "이론 영상 안 봐도 **벌써 진행이 생겼네**" | `/onboarding` → `/studio` | F-34, F-07, F-08, F-09, F-24 |
| 5. 스튜디오 진입 | **작곡 스튜디오**로 넘어가 **색 블록 타임라인**에 지금까지 고른 코드가 보인다. **재생 가능한 블록**만 있고 코드표 텍스트 목록은 없다 | "내가 만든 **줄**이 보인다 — DAW보다 단순하다" | `/studio` | F-19, F-41, F-42 |
| 6. 추천으로 이어 붙이기 | 타임라인 끝 **"+"** 또는 **다음 코드 추천 패널**에서 후보 **▶ 미리듣기** → **"에너지 올릴 때"** 문구를 보고 후보를 탭해 추가한다. 붙인 직후 **마지막 2코드**가 자동 재생된다 | "**들으면서** 고르니까 친구한테 물어볼 필요가 없다" | `/studio` | F-07, F-08, F-09, F-10, F-12, F-20, F-29 |
| 7. 마음에 안 들면 수정 | 마음에 안 드는 선택은 **Undo**로 되돌리거나, 블록을 **드래그**해 순서를 바꾼다. 바꾸면 **즉시 미리듣기**가 돌아간다 | "실수해도 **부담 없이** 다시 해볼 수 있겠다" | `/studio` | F-33, F-21, F-22, F-02 |
| 8. (선택) 이론 살짝 보기 | 궁금하면 후보 카드 **ℹ️ "왜?"** 를 펼쳐 cadence 등 **짧은 설명**을 본다. 기본은 **느낌 문구**만 본다 | "이론은 **나중에** 봐도 되겠다" | `/studio` | F-11, F-06 |
| 9. (선택) 더 넓게 탐색 | 추천만으로 부족하면 **"전체 코드 열기"** → **코드 그리드**에서 색·태그 보며 탭하거나, **유명 진행 프리셋**을 불러와 **코드 하나씩** 바꿔 본 뒤 스튜디오로 돌아온다 | "추천에 **갇히지 않고** 실험할 수 있다" | `/explore` → `/studio` | F-13, F-14, F-15, F-16, F-17, F-18 |
| 10. 4마디 완성 | **4번째 코드**가 타임라인에 추가되면 **"4마디 완성!"** 배지와 메시지가 뜨고 **자동 Play**된다. (선택) **4마디 구간 피드백**을 확인한다 | "**첫 진행을 만들었다!**" — Step 03 완성 경험 (Wow) | `/studio` | F-36, F-24, F-40 |
