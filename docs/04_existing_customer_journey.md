# Step 04. 기존 고객 여정

> 목적: `docs/03`에서 정의한 서비스 목적을, **지금** 사용자가 어떤 방식으로 달성하는지 정리한다.

> **Guide:** **우리가 만들 서비스의 여정이 아니다.** (우리 서비스 여정은 Step 11 `docs/11_customer_journey_map.md`) 유사 서비스·대체제·우회 방식마다 **여정을 하나씩** 두고, 단계별 사용자 행동·생각/감정을 표로 정리한다. 각 단계는 상세하게 작성한다. 이 내용을 토대로 Step 05에서 Pain/Wow 인사이트를 추출한다.

> **작성 메모:** Step 03 타겟(완전 초보, DAW·코드 모름, 밴드 첫 자작곡·취미) 기준으로 대표 여정을 합성해 작성함. 팀원 실경험과 다르면 bullet·행 추가·수정.

## 여정 1

> YouTube·검색 — "작곡 초보", "코드 진행", "화성학" 강의·튜토리얼

| 단계 | 사용자 행동 | 생각/감정 |
|---|---|---|
| 1. 동기 | 밴드에서 "우리도 자작곡 해보자" / 혼자 멜로디·가사만 떠올림 | "작곡하고 싶은데 뭐부터 해야 하지?" 기대 + 막막함 |
| 2. 검색 | YouTube·네이버에 "작곡 초보", "쉬운 코드 진행", "화성학 입문" 검색 | "무료로 배울 수 있겠다" 희망 |
| 3. 영상 선택 | 조회수 많은 10~20분 입문 영상, 재생목록 "3일 만에 화성학" 등 클릭 | "이거면 따라 할 수 있겠지" |
| 4. 시청·메모 | Cmaj7, I-IV-V-vi, 전조, 세븐스 코드 설명 듣고 필기 | "기호가 너무 많다" "멈추고 되감기 반복" "용어 사전이 필요한 것 같다" |
| 5. 따라하기 시도 | 영상 속 코드를 기타·키보드 없이 머릿속으로만 따라가거나, 악기 있는 친구에게 전달 | "내가 직접 확인할 수가 없다" "보컬만 하는 나랑은 거리가 있다" |
| 6. 이탈 | 영상 2~3개 더 보다가 피로, 또는 북마크만 해두고 종료 | "이론부터 다 배워야 하나" "당장 곡은 안 만들어지네" 좌절 |
| 7. 결과 | 작곡은 미루고, 기존 커버곡만 부르거나 밴드에 "아직 준비 중" | "나중에 시간 있으면" — 목표는 유지, 실행은 정체 |

## 여정 2

> 밴드·지인에게 물어보기 — 기타/키보드 친구, 선배에게 코드·진행 받기

| 단계 | 사용자 행동 | 생각/감정 |
|---|---|---|
| 1. 동기 | 자작곡 필요성(공연·학교 축제·밴드 내 역할) | "우리 밴드에 작곡하는 사람이 없으면 내가?" 부담 |
| 2. 요청 | 기타/키보드 친구에게 "코드 좀 짜줘", "이 분위기면 뭐 써?" DM·대면 요청 | "친구한테 맡기면 될 것 같다" 안도 |
| 3. 설명 시도 | "슬프지만 희망적인 느낌" 등 막연한 분위기만 전달 | "내가 원하는 걸 말로 설명하기 어렵다" |
| 4. 코드 전달 | 친구가 Am-F-C-G 등 적어 주거나, 카톡으로 코드표·사진 전송 | "글자는 알겠는데 소리가 상상 안 된다" |
| 5. 확인 | 친구가 기타로 한 번 쳐 주거나, 밴드 합주 때 들음 | "오 이 느낌!" (Wow) / "이게 맞나?" — 본인은 선택·수정 못 함 |
| 6. 수정 요청 | "조금 더 밝게" 등 재요청 → 친구 일정·기분에 따라 반응 달라짐 | "계속 부탁하기 미안하다" "내 곡이 아닌 것 같다" |
| 7. 결과 | 친구 손에 의존한 1곡 완성 또는, 친구 바쁘면 무기한 보류 | "작곡은 친구 있는 사람만 하는구나" 소외감 또는 "일단 한 곡은 했다" |

## 여정 3

> DAW·작곡 앱 직접 깔아보기 — GarageBand, BandLab, FL Studio Mobile 등

| 단계 | 사용자 행동 | 생각/감정 |
|---|---|---|
| 1. 동기 | "앱으로 작곡한다"는 말을 듣고 스토어에서 "작곡", "음악 만들기" 검색 | "앱 하나면 되겠지" |
| 2. 설치·첫 실행 | GarageBand/BandLab 설치, 튜토리얼 스킵 또는 짧게 클릭 | "화면이 많다" "트랙, MIDI, BPM이 뭐지" |
| 3. 루프·샘플 | 기본 제공 루프 드래그, 프리셋 비트 조합 | "만들기는 쉬운데 **내 멜로디·내 코드**가 아니다" |
| 4. 코드 입력 시도 | 코드 트랙·피아노 롤 찾다가 포기, 또는 무작위 건반 클릭 | "어디를 눌러야 코드가 되는지 모르겠다" |
| 5. 검색·도움말 | "GarageBand 코드 추가" 검색 → 영상·매뉴얼 (여정 1과 유사) | "앱 + 이론 둘 다 필요하구나" |
| 6. 이탈 | 앱은 깔려 있지만 열지 않거나, 루프곡만 잠깐 만들어 공유 | "DAW는 나중에" "작곡 앱 = 비트 만들기?" 인식 고착 |
| 7. 결과 | 완성된 **자작 코드 진행** 없음, 또는 템플릿 수준 산출물 | "진짜 작곡은 다른 레벨" 자신감 하락 |

## 여정 4

> 화성학·작곡 교재·온라인 강의 — 책, 패스트캠퍼스·클래스101, 대학 부전공 등

| 단계 | 사용자 행동 | 생각/감정 |
|---|---|---|
| 1. 동기 | "제대로 배우면 평생 쓸 수 있겠다" — 체계 학습 선택 | "시간 들여도 투자할 가치 있다" |
| 2. 선택 | 입문서(예: 화성학 개론), 또는 N주 완성 강의 결제 | "비용·시간 괜찮다" (또는 "공짜만 찾다가 포기") |
| 3. 학습 | 장조·단조, 다이아토닉 코드, cadence 단원 순차 수강 | "이해는 되는데 **귀와 손이 안 따라온다**" |
| 4. 연습 | 과제로 진행 4마디 작성 — 악보·코드표에 손으로 적기 | "채점 기준은 맞는데 **듣기**가 약하다" |
| 5. 실전 괴리 | 밴드에서 "이번 주에 데모 내자" — 이론과 일정 충돌 | "아직 3강인데 곡은 언제?" |
| 6. 이탈 | 강의 중도 이탈, 책 1/3만 읽고 책장 | "작곡은 **공부 끝난 사람**만 하는 줄 알았다" |
| 7. 결과 | 이론 조각은 알지만 **첫 자작곡 미완** | "언젠가 다 배우면…" 장기 연기 |

## 여정 5

> 기존 곡 코드·진행 베끼·변형 — 좋아하는 곡 코드표 보고 살짝 바꿔 쓰기

| 단계 | 사용자 행동 | 생각/감정 |
|---|---|---|
| 1. 동기 | "처음부터 짜기 어렵다" — 익숙한 곡에서 출발 | "이건 현실적인 방법" |
| 2. 코드표 검색 | Ultimate Guitar, 악보 사이트, "○○곡 코드" 검색 | "탭·코드표 처음 본다" "Capo가 뭐지" |
| 3. 연주·청취 | 친구가 치거나 원곡 들으며 진행 따라감 | "이 진행 **들으면** 알겠는데 **이름**은 모르겠다" (Step 03 Wow 단서) |
| 4. 변형 시도 | 마지막 코드 하나 바꿔 봄 → 어색함 | "바꾸면 왜 이상한지 설명 못 함" |
| 5. 한계 인식 | 멜로디·가사는 새로 쓰려 해도 **화성은 원곡 의존** | "자작곡이라 부르기 애매하다" |
| 6. 결과 | 커버+α 수준, 또는 표절 걱정으로 중단 | "베끼기 말고 **직접 고르고 싶다**" — Step 03 니즈와 연결 |

> **범위 메모:** Step 03에서 "기존 곡 코드 **이해·분석**은 우리 서비스 범위 밖"이지만, **베끼기·변형**은 초보가 실제로 쓰는 **대체 여정**이므로 Step 04·05용으로 포함.

## 여정 6

> AI 챗봇·생성 AI — ChatGPT 등에 "슬픈 코드 진행 4마디" 요청

| 단계 | 사용자 행동 | 생각/감정 |
|---|---|---|
| 1. 동기 | SNS·친구에게 "GPT한테 코드 받았다" 얘기 듣음 | "요즘은 AI로 되겠다" |
| 2. 프롬프트 | "밝은 코드 진행 8마디", "발라드 느낌 Am으로 시작" 입력 | "말로 요청하니까 쉽다" |
| 3. 결과 수신 | Am-F-C-G, Roman numeral, 짧은 설명 텍스트 | "글로는 와닿는데 **소리가 없다**" (Step 03 Pain) |
| 4. 확인 시도 | 피아노 앱·친구에게 전달해 들음 | "AI가 준 게 **맞는지** 내 귀로 판단하기 어렵다" |
| 5. 수정 루프 | "2번째 코드 더 슬프게" 재프롬프트 — 맥락 유지 불안정 | "대화로 **한 진행을 다듬기** 번거롭다" |
| 6. 결과 | 텍스트 코드표만 있거나, 만족 못 하고 다른 여정(1·2)으로 회귀 | "AI + **즉시 재생·시각화**면 좋겠다" — 우리 서비스 방향 힌트 |

## 여정 7

> 포기·미루기 — "작곡은 재능 있는 사람만"

| 단계 | 사용자 행동 | 생각/감정 |
|---|---|---|
| 1. 동기 | 밴드·취미로 작곡 욕구는 있음 | "하고 싶다" |
| 2. 1~2회 시도 | 위 여정(1·3·4 등) 중 하나 짧게 시도 | "생각보다 어렵다" |
| 3. 자기귀인 | "난 보컬만 하면 되지", "작곡은 ○○(친구) 재능" | "내 영역이 아니다" |
| 4. 대체 역할 | 커버곡 선곡·보컬만 담당, 작곡 얘기 회피 | "밴드 안에서 **작곡 못 하는 사람**" 고정 |
| 5. 간헐적 재시도 | 방학·새 학기마다 "이번엔 해볼까" → 동일 패턴 | "시간 없어서"가 아니라 **진입장벽** |
| 6. 결과 | 자작곡 0곡, Step 03 목표(첫 완성 경험) **미달성** | 잠재 Pain: "시작조차 못 했다" |
