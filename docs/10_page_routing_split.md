# Step 10. 기능을 페이지로 나누기

> 목적: Step 09에서 모은 **기능을 화면(페이지) 단위로 나눠** 배치한다.

> **Guide:**  
> 1. Agent가 `docs/09_feature_list.md`의 기능 목록을 읽고 **페이지 구성 초안**을 작성한다.  
> 2. 각 페이지에 **어떤 기능(F-xx)** 이 들어가는지 나열한다.  
> 3. 팀이 페이지 수·이름·구성을 보며 수정한다.  
> - Step 09의 기능은 **어느 페이지에든 모두 배치**한다. (이 단계에서 기능을 빼지 않는다)  
> - 사용자 흐름이 자연스럽게 이어지도록 페이지를 묶는다. (예: 로그인 → 메인 → 상세)  
> - 한 페이지에 기능이 너무 많으면 **페이지를 나눈다.**  
> - **우선순위·스펙 아웃(이번에 만들지 않을 기능)** 은 Step 13 `docs/13_priority_matrix.md`에서 정한다.

**프롬프트 예시**

```
docs/09_feature_list.md와 docs/03~08 맥락을 읽고 docs/10_page_routing_split.md를 채워줘.

규칙:
- Step 09의 모든 기능(F-xx)을 어느 페이지에든 배치 (누락 없음)
- 페이지마다 이름·경로·목적·포함 기능 목록을 작성
- 기능 ID(F-xx)와 기능명을 함께 적기
- 해커톤 MVP 기준으로 페이지 수는 과하지 않게 (보통 3~6개)
- "페이지 간 이동 흐름" 섹션도 채우기
- 구조와 상단 Guide는 유지
```

## 페이지 1: 온보딩

> **경로:** `/onboarding`  
> **목적:** 첫 방문자가 **회원가입·이론 없이** 3스텝만으로 분위기와 시작 코드를 고르고, 작곡 스튜디오로 넘어가게 한다.

- **F-03** 분위기 칩 선택 (밝은 / 슬픈 / 몽환적 / 긴장감 등)
- **F-04** 분위기 칩 선택 시 해당 분위기 **시작 코드 3~4개** 카드 제시
- **F-05** 코드 카드 **한국어 느낌 라벨** 표시 (기호 대신)
- **F-34** **온보딩 3스텝** (분위기 → 시작 코드 → 추천 2~3회 → Play)
- **F-35** **온보딩**에서 회원가입·DAW 용어 설명 **생략**

## 페이지 2: 작곡 스튜디오 (메인)

> **경로:** `/studio`  
> **목적:** **듣고 고르는 작곡 놀이터** — 타임라인에 코드 진행을 만들고, 추천·재생·설정·완성까지 **핵심 작업**을 한다.

- **F-01** 코드 블록·후보 카드 **클릭·탭 시 즉시** 해당 코드 재생
- **F-02** 타임라인에 코드 **드래그·드롭 추가** 시 **1마디 자동 재생**
- **F-06** **코드명(Am, Cmaj7) 표시** 고급 모드 **토글** (기본 숨김)
- **F-07** **"다음 코드 추천" 패널** — 후보 **3~4개** 표시
- **F-08** 추천 후보마다 **▶ 미리듣기** (직전 코드 → 후보 2코드)
- **F-09** 추천 후보 **클릭 시 타임라인 추가** + **마지막 2코드** 자동 재생
- **F-10** 추천 후보 **"이럴 때 고르면 좋다"** 한 문장 안내
- **F-11** 추천 후보 **ℹ️ "왜?"** 접기 — 이론 설명 (선택 노출)
- **F-12** **화성학 규칙 기반** 다음 코드 추천
- **F-13** **"전체 코드 열기"** 버튼 → `/explore` 이동
- **F-19** 가로 **코드 타임라인** — 분위기별 **색 블록**
- **F-20** 블록 **사이 "+"** 로 다음 코드 끼워 넣기
- **F-21** 타임라인 블록 **드래그로 순서 변경**
- **F-22** 순서 변경 후 **바뀐 순서로 즉시 미리듣기**
- **F-23** 블록 **길이(마디 단위)** 조절
- **F-24** **▶ Play / ■ Stop** — 진행 **처음부터 끝까지** 순차 재생
- **F-25** 재생 중 **현재 코드 블록 하이라이트**
- **F-26** **BPM 조절** (80~120, 기본 90) · **4/4** 박자
- **F-27** **마디 수 슬라이더** (4 / 8마디 등)
- **F-28** **코드당 박 수 슬라이더**
- **F-29** **"이렇게 이어지면 이런 느낌"** — 연결 시 분위기·감각 안내
- **F-30** Play 시 **기본 드럼·베이스 패턴** 함께 재생
- **F-31** **내장 피아노·기타 MIDI 톤**으로 코드 재생
- **F-32** **모바일·태블릿** **큰 카드 UI** (한 손 조작)
- **F-33** **되돌리기(Undo)** — 직전 코드 선택 취소
- **F-36** **4번째 코드 추가 시 "4마디 완성" 배지** + 자동 Play + 완성 메시지
- **F-37** **자연어·분위기 입력** (선택) — 상단 입력창
- **F-38** 자연어 입력 결과 → **타임라인 블록 + 즉시 Play**
- **F-40** **4마디 구간 전체** 좋고 나쁨 **피드백**
- **F-41** **인터랙티브 시각화**로 코드 진행 **탐색·실험** (타임라인·블록 중심)
- **F-42** UI **텍스트 코드표 목록 없음** — **재생 가능한 블록**만 표시

## 페이지 3: 코드·진행 탐색

> **경로:** `/explore`  
> **목적:** 추천 **밖**에서 **전체 코드·유명 진행**을 훑고, 골라 스튜디오 타임라인으로 가져온다.

- **F-14** **24 diatonic 코드 그리드** — 색·분위기 태그와 함께 표시
- **F-15** 그리드 코드 **탭 → 즉시 재생 + 타임라인 추가** (추가 후 `/studio` 복귀)
- **F-16** **"유명 진행 프리셋"** 목록 (카논 진행, 4코드 팝 등)
- **F-17** 프리셋 선택 시 타임라인에 **진행 통째로 불러오기** (`/studio`로 전달)
- **F-18** 불러온 프리셋에서 **마디·코드 하나씩 교체** 실험 (`/studio`에서 편집)

## 페이지 4: 데모 공유

> **경로:** `/share`  
> **목적:** 만든 코드 진행을 **밴드·동아리**에 들려줄 **데모(mp3·링크)** 를 만든다.

- **F-39** **"데모 공유"** — **30초 mp3** 또는 **공유 링크** 생성

## 페이지 간 이동 흐름

> 사용자가 페이지를 어떻게 오가는지 순서대로 적는다.

1. **첫 방문** → `/onboarding` — 분위기 칩(F-03) → 시작 코드 카드 탭(F-04, F-05) → 온보딩 3스텝(F-34, F-35) 완료
2. **온보딩 완료** → `/studio` — 선택한 시작 코드가 타임라인(F-19)에 채워진 상태로 진입
3. **스튜디오에서 작곡** — 추천 패널(F-07~12)로 코드 추가(F-09) → Play(F-24) · Undo(F-33) · 4마디 배지(F-36)
4. **더 넓게 고르고 싶을 때** — 스튜디오 **"전체 코드 열기"**(F-13) → `/explore` → 그리드(F-14, F-15) 또는 프리셋(F-16~18) 선택 → `/studio` 복귀
5. **데모 들려주기** — 스튜디오 **"데모 공유"** → `/share`(F-39) → mp3·링크 생성 후 밴드 단톡 등 공유
6. **재방문** — `/onboarding` 건너뛰고 **`/studio` 바로** (로컬 저장 또는 마지막 진행 복원 — 구현은 Step 13 이후)
