# Step 09. 기능 나열

> 목적: 지금까지 논의에서 나온 **기능 아이디어를 하나도 빠뜨리지 않고** 모은다.

> **Guide:**  
> 1. Agent가 `docs/03`~`docs/08`을 읽고, 언급된 **모든 기능**을 추출한다.  
> 2. **순서·중요도·선후 관계 없이** 일렬로 나열한다.  
> 3. 팀이 빠진 기능이 있으면 **팀이 추가한 기능**에 적는다.  
> - 기능은 **사용자가 할 수 있는 일** 또는 **서비스가 제공하는 것** 단위로 쓴다. (예: 로그인, 일정 자동 추천, 알림 보내기)  
> - 큰 덩어리(예: "앱 전체")로만 적혀 있어도 **일단 포함**하고, 안에 여러 기능이 섞여 있으면 **풀어서** 각각 적는다.  
> - 비슷한 기능이라도 **논의에서 따로 나왔으면 각각** 적는다.  
> - **이 단계에서 하지 않는 것:** 우선순위 정하기, 중요도 판단, MVP만 골라내기, 제외·합치기  
> - 우선순위·제외는 **Step 13**에서 한다. 여기서는 **놓치지 않고 모으는 것**만 한다.

**프롬프트 예시**

```
docs/03~08을 읽고 docs/09_feature_list.md의 "Agent가 추출한 기능" 섹션만 채워줘.

규칙:
- docs/03~08 어디에든 나온 기능·아이디어를 빠짐없이 포함 (목적·가치, 여정, Pain/Wow, 가설, 키워드, 질문·답변 등)
- F-01부터 번호를 매겨 일렬로 나열 (번호·순서는 우선순위가 아님)
- 기능명은 짧고 구체적으로 (한 줄)
- 우선순위 판단·제외·MVP 선별 하지 않는다. 의심스러워도 일단 나열
- "팀이 추가한 기능" 섹션은 건드리지 않는다.
- 구조와 상단 Guide는 유지
```

## Agent가 추출한 기능

> `docs/03`~`docs/08`에서 나온 기능. **빠짐없이** 나열. Agent가 채운다.

- **F-01** 코드 블록·후보 카드 **클릭·탭 시 즉시** 해당 코드 재생 (Web Audio / Tone.js)
- **F-02** 타임라인에 코드 **드래그·드롭 추가** 시 놓는 즉시 **1마디 자동 재생**
- **F-03** **분위기 칩** 선택 (밝은 / 슬픈 / 몽환적 / 긴장감 등)
- **F-04** 분위기 칩 선택 시 해당 분위기 **시작 코드 3~4개** 카드 제시
- **F-05** 코드 카드 **한국어 느낌 라벨** 표시 (예: "잔잔한 시작", "희망적인 시작" — 기호 대신)
- **F-06** **코드명(Am, Cmaj7) 표시** 고급 모드 **토글** (기본 숨김)
- **F-07** **"다음 코드 추천" 패널** — 마지막 코드 기준 후보 **3~4개** 표시
- **F-08** 추천 후보마다 **▶ 미리듣기** (직전 코드 → 후보 **2코드** 짧게 이어 재생)
- **F-09** 추천 후보 **클릭 시 타임라인에 추가** + 추가 직후 **마지막 2코드** 자동 재생
- **F-10** 추천 후보 카드 아래 **"이럴 때 고르면 좋다"** 한 문장 안내 (20자 이내)
- **F-11** 추천 후보 **ℹ️ "왜?"** 접기 — cadence·dominant 등 **이론 설명** (선택 노출)
- **F-12** **화성학 규칙 기반** 다음 코드 추천 (다이아토닉·cadence 등)
- **F-13** **"전체 코드 열기"** 버튼 — 추천 밖 코드 탐색
- **F-14** **24 diatonic 코드 그리드** — 색·분위기 태그(밝음/어두움)와 함께 표시
- **F-15** 그리드 코드 **탭 → 즉시 재생 + 타임라인 추가**
- **F-16** **"유명 진행 프리셋"** 목록 (카논 진행, 4코드 팝 등)
- **F-17** 프리셋 선택 시 타임라인에 **진행 통째로 불러오기**
- **F-18** 불러온 프리셋에서 **마디·코드 하나씩 교체** 실험
- **F-19** 가로 **코드 타임라인** — 분위기별 **색 블록**으로 진행 표시
- **F-20** 블록 **사이 "+"** 로 다음 코드 끼워 넣기
- **F-21** 타임라인 블록 **드래그로 순서 변경**
- **F-22** 순서 변경 후 **바뀐 순서로 즉시 미리듣기** 재생
- **F-23** 블록 **길이(마디 단위)** 조절
- **F-24** 상단 **▶ Play / ■ Stop** — 진행 **처음부터 끝까지** 순차 재생
- **F-25** 재생 중 **현재 코드 블록 하이라이트**
- **F-26** **BPM 조절** (80~120, 기본 90) · **4/4** 박자
- **F-27** **마디 수 슬라이더** (4 / 8마디 등)
- **F-28** **코드당 박 수 슬라이더** (1코드=1마디 vs 2박 등)
- **F-29** **"이렇게 이어지면 이런 느낌"** — 코드 연결 시 **분위기·감각** 안내 (Step 03 핵심 가치)
- **F-30** Play 시 **기본 드럼·베이스 패턴** 함께 재생 (간단 4/4 반주)
- **F-31** **내장 피아노·기타 MIDI 톤**으로 코드 재생 (악기 입력 없음)
- **F-32** **모바일·태블릿** 대응 **큰 카드 UI** (한 손 조작)
- **F-33** **되돌리기(Undo)** — 직전 코드 선택 취소
- **F-34** **온보딩 3스텝** (분위기 → 시작 코드 → 추천 2~3회 → Play)
- **F-35** **온보딩**에서 회원가입·DAW 용어 설명 **생략**
- **F-36** **4번째 코드 추가 시 "4마디 완성" 배지** + 자동 Play + 완성 메시지
- **F-37** **자연어·분위기 입력** (선택) — "밝은 코드 진행 8마디" 등
- **F-38** 자연어 입력 결과를 **타임라인 블록 + 즉시 Play**로 반환 (텍스트 코드표만 X)
- **F-39** **"데모 공유"** — **30초 mp3** 또는 **공유 링크** 생성
- **F-40** **4마디 구간 전체** 좋고 나쁨 **피드백** (Step 03 — 형태·우선순위 미정, 일단 포함)
- **F-41** **인터랙티브 시각화**로 코드 진행 **탐색·실험** (Step 03 — 시각 탐색 전반)
- **F-42** UI 기본 화면 **텍스트 코드표 목록 없음** — **재생 가능한 블록**만 표시
